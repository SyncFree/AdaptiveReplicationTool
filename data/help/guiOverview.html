GUI

The GUI shows the progess of the replication and the messages logged.

The tool is composed of two views:

-Emulation View
-Log View

The Emulation View

This view is located a the top of the GUI tool and display the DC, the data they hold and their state. Also the control to run tests if any is set.

A DC is represented with a cylinder image (this can be changed in the configuration file), with the name of the DC at the top and the value of the replica on that DC, if a replica exist in the DC. The cylinder image changes to give an indication of the existence of a replica and the replica strength in the DC.

The tool already has an implementation of the proposed Adaptive Replication algorithm, called here Ant Modifier. Other algorithms may be implemented and run by specifying one in the configuration file.

The tool support two ways of running it:

-By assigning to each DC a read and a write key respectively that when pressed will send an appropriate request to the that DC.
- By setting tests in the configuration file, in which case the GUI will display a test control view with buttons for 'play', 'pause', 'next', and 'stop' the test, which are called here states. Also a st of test may be repeated for a number of times. The test will be started on the state specified in the configuration file.
-Also an individual request may, in a test set, have associated with it an action (set a valid state) for an specific repetition index.

The Log View

This view is placed at the bottom of the GUI and it is composed of two sub-views. The log controls menu view that allows to close, clear, help which provides simple test help, and filter the log view.

The main log view shows the different log messages posted.
